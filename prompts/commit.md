---
description: Commit all changes in clean, small conventional commits
---
Review all current changes (`git diff` and `git status`). Then commit them in clean, small, logical commits using the **Conventional Commits** format (`feat:`, `fix:`, `refactor:`, `chore:`, `docs:`, `style:`, `test:`, etc.).

Rules:
- Each commit should be a single logical unit of change
- Stage and commit related files together, not everything at once
- Write concise, meaningful commit messages in English
- Use the correct conventional commit type and scope where appropriate
- If there are unrelated changes, split them into separate commits
- Do NOT bundle unrelated changes into one commit
